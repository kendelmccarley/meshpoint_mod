**Meshpoint v0.6.0** is out. Your Meshpoint can now send and receive messages on the mesh. Full chat UI, radio configuration, node discovery, and a dashboard overhaul.

**Native mesh messaging(!!!!!). **Send and receive Meshtastic messages directly from your browser. Broadcast to LongFast, talk on custom channels, DM individual nodes. MeshCore messaging works through the USB companion. The SX1302 transmits with the correct Meshtastic sync word and encryption: phones and nodes see your Meshpoint as a regular participant on the mesh. No phone needed.

**Full chat UI. **Conversations organized by channel and contact in a sidebar. Signal info (SNR, RSSI) on every received bubble so you can see link quality at a glance. Duplicate badge shows how many times a relayed message was heard. Message history persisted locally.

**Radio config from the dashboard. **No more SSH to change settings. Region, modem preset (LongFast, MediumFast, etc.), frequency override, TX power, and duty cycle are all configurable from the Radio tab. Add and remove custom channels with PSKs. Toggle TX on/off. Everything saves to local.yaml and survives restarts.

**Node discovery. **Live node cards for every node your Meshpoint has heard: name, ID, protocol, hardware model, signal strength, battery, last seen. Click any node to open a detail drawer with signal history. DM a node directly from its card.

**Dashboard overhaul. **New messaging tab, node cards grid, radio settings page. Packet feed now shows frequency and spreading factor columns. The whole UI got a refresh.

**CLI operational report. **New `meshpoint report` command gives you a full-screen operational dashboard in the terminal: RX stats, traffic breakdown, signal averages, system metrics, and composite health status.

**Setup wizard improvements. **Each Meshpoint now gets a unique random Meshtastic node ID during setup (no more collisions). MeshCore companion configuration is its own wizard step.

**New install path** (fresh SD card):
```
sudo git clone https://github.com/KMX415/meshpoint.git /opt/meshpoint
sudo bash /opt/meshpoint/scripts/install.sh
sudo reboot
sudo meshpoint setup
```
TX works out of the box. No extra steps.

**Updating from a previous version** (one-time extra steps for v0.6.0):
```
cd /opt/meshpoint
sudo git pull origin main
sudo bash /opt/meshpoint/scripts/patch_hal.sh
sudo cp config/sudoers-meshpoint /etc/sudoers.d/meshpoint
sudo chmod 440 /etc/sudoers.d/meshpoint
sudo cp scripts/meshpoint.service /etc/systemd/system/meshpoint.service
sudo systemctl daemon-reload
sudo systemctl restart meshpoint
```
`patch_hal.sh` patches the concentrator for Meshtastic TX and recompiles (about 2 minutes). The other commands set up permissions for dashboard config saves. All one-time: future updates go back to `git pull` + `restart`.

Tested on RAK Hotspot V2 and SenseCap M1 hardware. Fresh install and update path both validated.
